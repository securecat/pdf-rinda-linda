# PDFリンダリンダ

視覚障害者など目の不自由な方は、PDFを読むのを諦めざるを得ない状況にあります。それは、世間に現存するPDFのほとんどが、アクセシビリティを考慮していないからです。具体的には、以下のような問題が挙げられます。

- Altエントリが無い
- それどころかテキストレイヤーが無い
- 見た目の文書構造どおりに読み上げられない
- 同一文書構造のPDF文書群であっても、読み上げられ方がばらばらである

「PDFリンダリンダ」は、そうした現実の問題に対処すべく以下の目的遂行のために開発しています。

- どのようなPDFであっても内容を読み解き、見た目の文書構造を再現したMarkdownを出力してスクリーンリーダー等でのPDF読み上げを支援します
- Markdown出力にかかったコストを可視化し、当該PDFの著者に対して、ただPDFを読みたいだけの人たちがどれほどの労力をかけなければ読めないのかという事実を伝え、アクセシブルなPDFづくりを啓発します

## 名称に込めた想い

PDFリーダー界のリーダーを標榜して「PDFリーダーリーダー」と名付けたものの、何回も口にしているうちに段々PDFリンダリンダの発音になってしまい、結果「PDFリンダリンダ」を正式名称としました。

## 機能

### 現在の機能（v0.x）

- ChromeでPDFを開いているときにサイドパネルを表示し、ページ内容をMarkdownで出力する
- Claude APIを使ってページ画像から直接OCRを行う（日本語PDFに多いCMap問題を回避するため、テキストレイヤーは補助情報としてのみ使用）
- 横長ページ（見開きPDF等）を自動検出し、左右に分割して個別送信することで実効解像度を向上
- Popupからページ番号を指定して処理対象ページを選択できる
- 利用にはAnthropicのAPIキーが必要

### v1.0.0 に向けて開発中

**「1ページを、どんなレイアウト・内容でも、誰もが平等に読めるようにする」** をゴールに開発を続けています。

現状の単一プロンプト方式では、装飾の強いデザインページや多段組レイアウトで内容が欠落することがあります。v1.0.0 では、ページを視覚的なまとまりに分解してから個別に処理するマルチステップ方式に移行し、どのようなPDFページにも対応できる品質を目指します。

多ページの連続処理はv1.0.0以降の対応予定です。当面は1ページずつユーザーが指定して読む運用を前提とします。

## インストール

### Chrome ウェブストア

準備中（当面はデベロッパーモードでの開発のみとします）

### デベロッパーモード（手動インストール）

1. このリポジトリをダウンロードまたはクローン
2. Chromeで `chrome://extensions` を開く
3. 右上の **デベロッパーモード** を有効にする
4. **パッケージ化されていない拡張機能を読み込む** をクリックし、リポジトリのフォルダを選択
